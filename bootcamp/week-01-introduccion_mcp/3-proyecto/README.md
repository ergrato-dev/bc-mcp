# Proyecto Semana 01 — Mapa de Arquitectura MCP

## 🎯 Descripción

Crear un diagrama SVG completo y documentado de la arquitectura MCP para un caso de uso
real de tu elección (sistema de archivos, base de datos, API externa, etc.). Además,
analizar 3 casos de uso reales de MCP y dejar el entorno Docker funcionando.

## 📋 Instrucciones

1. Lee el material teórico de [`1-teoria/`](../1-teoria/README.md) antes de comenzar
2. Completa las prácticas de [`2-practicas/`](../2-practicas/README.md) como preparación
3. Trabaja en el directorio `starter/` — **no modifiques** archivos fuera de él
4. Usa Docker para levantar el entorno (`docker compose up`)
5. Verifica todos los entregables antes de la entrega

## 📌 Entregables

- [ ] `architecture-diagram.svg` — Diagrama SVG de la arquitectura MCP para tu caso de uso
- [ ] `use-cases.md` — Análisis documentado de 3 casos de uso reales de MCP
- [ ] `docker-compose.yml` descomentado y funcionando con MCP Inspector
- [ ] Respuestas completas en `architecture-analysis.md`

## 🏗️ Estructura del Proyecto

```
3-proyecto/
├── README.md          # Este archivo
└── starter/           # Tu trabajo va aquí
    ├── README.md      # Instrucciones de setup
    ├── architecture-diagram.svg   # TODO: crear tu diagrama
    ├── architecture-analysis.md  # TODO: completar el análisis
    ├── use-cases.md               # TODO: documentar 3 casos de uso
    └── docker-compose.yml         # TODO: descomentar para levantar el entorno
```

> ⚠️ La carpeta `solution/` es solo para instructores y está en `.gitignore`.
